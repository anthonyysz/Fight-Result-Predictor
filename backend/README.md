# Backend

The back end gives us all of the functions we need to properly run everything. Running scrapers, making predictions and loading data to the database keeps everything organized and running smoothly.

## 1. Layout

- `src/historical_scraper/`
  - Scraps past UFC events, starting at a certain date through present day
- `src/upcoming_scraper/`
  - Scrapes the next upcoming event only
- `src/model_training/`
  - Retrains the models and saves the new ones, reusing code from notebook 3
- `src/api/`
  - Runs the scrapers, loads the data into the database, and makes predictions
- `data/reference/`
  - Source files the scraper reads, such as `fighter_aliases.csv` and `rankings_history.csv`
- `data/generated/`
  - CSV outputs generated by the scraper
- `db/`
  - PostgreSQL schema and test queries
- `.env`
  - Backend environment variables and database connection settings

## 2. Install

From the repo root:

```powershell
.venv\Scripts\pip install -e .\backend
```
## 3. Load the back end

From backend directory:
```powershell
Invoke-WebRequest -Uri http://127.0.0.1:8000/health -UseBasicParsing
```

To start it:
```powershell
$env:PYTHONPATH = (Resolve-Path .\src).Path
```

To clear it:
```powershell
Stop-Process -Id 29604
```

To reload it:
```powershell
& "C:\Users\Anthony\GitHub\fight-result-predictor\.venv\Scripts\python.exe" -m uvicorn api.app:app --host 127.0.0.1 --port 8000 --reload
```

## 4. Database testing

To get to the database from powershell:
```powershell
psql $env:DATABASE_URL
```

```powershell
psql $env:DATABASE_URL -c "SELECT fight_date, red_fighter, blue_fighter, weight_class, source_name FROM public.all_fights ORDER BY fight_date DESC LIMIT 20;"
```

## 5. Running Programs

Retrain models:
```powershell
.venv\Scripts\python -m model_training.retrain_models
```

Use historical scraper:
```powershell
& "C:\Users\Anthony\GitHub\fight-result-predictor\.venv\Scripts\python.exe" -m historical_scraper.main --start-date 2024-12-14
```

Use upcoming scraper:
```powershell
& "C:\Users\Anthony\GitHub\fight-result-predictor\.venv\Scripts\python.exe" -m upcoming_scraper.main
```

## 6. Loading data to RDS

To load historical scraped data into RDS:
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/fights/load' -ContentType "application/json" -Body '{"source":"recent"}'
```

Load from testing.csv database
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/fights/load' -ContentType 'application/json' -Body '{"source":"testing"}'
```

Load upcoming fights
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/fights/load' -ContentType 'application/json' -Body '{"source":"upcoming_fights"}'
```

Load upcoming metadata
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/fights/load' -ContentType 'application/json' -Body '{"source":"upcoming_metadata"}'
```

## 7. Generate Predictions and complete finished fights

Generate upcoming predictions from RDS using S3-hosted models
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/upcoming-predictions/generate'
```

Complete fights after they are done
```powershell
Invoke-RestMethod -Method Post -Uri 'http://127.0.0.1:8000/admin/upcoming-fights/finish'
```
